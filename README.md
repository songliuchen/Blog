1、assets 文件夹内容介绍
主要存放一些资源文件，和通过样式文件，包括css，img文件夹，有需要可添加
    a、css文件夹内容介绍
        目前css 文件夹中只有一个css 文件，将通用的样式设置放到common 中，例如去除li 的圆点样式等，其他具体页面样式不要放到通用样式文件中
    b、img 文件夹存放素有图标文件
    c、其他文件，超过两个创建文件夹单独处理，就一个则直接放到根目录

2、components 文件夹内容介绍
主要存放vue模板，按业务功能分成不同的模块，对应不同的子文件夹，vue 的设计理念就是模板化

    a、common 文件夹提供通过组件模板
        1）何为通用组件模板，就是不止在一个页面使用的模板，或者可以通过对外提供参数即可完成不同数据加载的模板，只需要做微调的组件
        2）现包含模板有 顶部导航（topnav.vue）左侧文章导航（leftnav.vue），标签云导航（readmarklist.vue）、搜索条导航（searchbar.vue）

    b、其他文件夹对应具体菜单
        1）最外一级整体模板，不放具体文件夹下面，直接放在components 文件夹下，例如 index.vue 表示首页，articles.vue表示技术文章
        2）然后对应每个页面有自己的子文件夹，将页面内容拆分成具体组件，分块编写，然后在主页面组装，也就是1）中提到的文件中组装

3、module 文件夹内容
为实现多页面应用，配置webpack 程序入口使用，和components 文件夹一下，按业务分不同文件夹，文件夹命名与components 文件夹相同，问价命名也相同

4、其他文件夹暂时不用考虑


5、样式命名
样式命名必须先戴上自己的容器的样式，再写自己的样式，模板下的第一个容器的样式名就是文件名，例如articles.vue 模板下面的 <div class="articles"> 改文件中所有样式都必须以.articles 打头，避免样式名重复 例如 .articles .container

6、功能开发模式
    1）尽量低耦合，组件内部能搞定的事情，不要放到父组件或者子组件处理
    2）需要耦合的做成通用参数对外提供，逻辑还是组件内部处理
    3）组件内不不要涉及过多样式，组件只关心组件内部样式，不关心组件本身在其父组件的布局大小等
